query evolution description:
The user searches for companies in Croatia. The first query, which looks for organizations in Croatia times out. The couple of queries after that add a distinct to the output, or only search for results in Croatian, but keep timing out.
When the search changes from organizations to companies in Croatia, we start getting results. Then, the search is widened again to include results in English.

user story:

Query 1: The user searches for instances of (P31) subclasses of (P279*) organizations (http://www.wikidata.org/entity/Q43229) located in Croatia (http://www.wikidata.org/entity/Q224). The query times out.

Query 2: A condition of "Distinct" is added to the output. It still times out.

Query 3: "Distinct" is dropped and the search is restricted to results in Croatian. Still times out.

Query 4: An order by is added (ascending). Still times out.

Query 5: Language is changed to English. Still times out.

Query 6: "Organization" is changed to "Company" (http://www.wikidata.org/entity/Q783794), and the English language tag is removed. Now we get results, but the language tags are all over the place. (Looks like Cyrillic, Arabic, Bangla,...)

Query 7: Language tags are restricted to Croatian again.

Query 8: Language tags are changed to English. The query now returns an alphabetically ascending list of companies in Croatia, all results in English.
